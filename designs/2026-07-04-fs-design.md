# Slice D8 — `FS` (file status) — Synthesized design (FAITHFUL DENY)

**Date:** 2026-07-04 · **Track:** user-facing, full pipeline · **Decision:** FAITHFUL
DENY (user-chosen at the Stage-1/capture gate). On the shipped reference board the
seeded sec-255 sysop lacks `ACS_CONFERENCE_ACCOUNTING`, so **every `FS` form denies**
with `Command requires higher access.` (captured on every probe). The `fileStatus(0)`
accounting table is **not** built here — it is A11 / Tier-I territory.

Produced by the Stage-3 judge-panel (3 Fable framings → Opus judge/synthesize → Opus+
Fable adversarial refutation). Ranked totals **A 24 > C 23 > B 22**; base = Framing A
(minimal-change, the only capture-grounded candidate) with grafts from C (reuse the
`command_tokens` seam, Step-0 capture gate, boundary tests, docs corrections) and B
(reject a `CommandOutcome::NotAllowed` dispatcher enum; const-home justification).

**Two load-bearing objections resolved (round 0):**
1. **§10.4 premature abstraction — dropped `Right::ConferenceAccounting`.** The captured
   deny is reachable *without* any domain change; a constant-false gate with no
   `ACCESS.<level>` reader models the *outcome* while pretending to model the
   *mechanism*. The `FS` arm writes the deny **unconditionally**. Blast radius 4 → 2
   code files.
2. **Grammar row G6 reclassified** from "legacy-identical" to a **labelled tokenizer
   departure** (whitespace/tab folding differs from legacy).

**Two minor objections folded (round 1, both refuters clean):** **`FS` is NOT
advertised** — no `advertised_token => Some("FS")`, no `Conf02/Menu5.txt` FILES row.
The compiler forces only the `advertised_token` match arm's *existence*, not a `Some`
value; the reference board does not advertise `FS` (help-audit capture), and advertising
a command that denies 100% of users is the §10.4 "advertise-then-reject drift" class.
`advertised_token(FileStatus) => None`.

## 7. Ports / adapters — no new domain arm; the minimal deny

**No new port, adapter, store, domain enum, or `DispatchOutcome`/`NotAllowed` result
channel. The only new symbol is one wire const, `HIGHER_ACCESS_LINE`.**

**§10.4 — why no `Right::ConferenceAccounting`.** The captured deny is reachable with
existing seams: an unconditional `self.terminal.write(HIGHER_ACCESS_LINE).await?` in the
`FileStatus` arm reproduces every captured probe with zero domain change. A
`Right::ConferenceAccounting` arm would be a constant-false gate — false for every
constructible user, whose granted side is dead code — modelling the *outcome* while
pretending to model the *mechanism* (a config-resolved `checkSecurity`), yet NextExpress
has no `ACCESS.<level>` reader. The `EditConferenceFlags`/`CF` precedent is **not**
parallel (that gate is exercised in both directions); this one has a single reachable
outcome. The closer precedent is `ACS_JOIN_CONFERENCE` on `J`, **deliberately left
unported** (Stage-1 brief), and the deleted Silent `EchoMode`. Per §10.4 the strictly-
smaller no-Right option is the default; recorded at the gate (§10.10) rather than
silently chosen.

**Modelled behaviour.** Legacy `checkSecurity` resolves `ACS_CONFERENCE_ACCOUNTING` from
board configuration at check time (per-user override → `TOOLTYPE_DEFAULT_ACCESS` →
`Access/ACS.<level>`, `express.e:8455-8497`), never from the user record; the shipped
board grants it to nobody, sec 255 included (captured). The faithful analogue with no
captured granted path is a plain unconditional deny at the one command that reaches the
gate — not a per-user seed bit, not a domain enum, not an `ACCESS.<level>` reader.

**Option G — the named exit ramp.** When an A11/H-tier slice first captures a *granted*
`fileStatus(0)` path, that slice reintroduces the gate — `Right::ConferenceAccounting`
through `User::has_access`, or a `checkSecurity`/`ACCESS.<level>` reader — with **both
branches live and exercised**, and a `higher_access()` helper is the refactor to perform
when a **second** denying command lands.

**Rejected explicitly:** a `CommandOutcome::NotAllowed` dispatcher enum — one
`RESULT_NOT_ALLOWED` producer does not force a dispatcher-wide channel; the existing
inline-write pattern already covers it (§10.4, graft B).

```rust
// rust/src/app/menu_flow/mod.rs — const beside UNKNOWN_COMMAND_LINE (:123)
/// `higherAccess()` (`amiexpress/express.e:3038`,
/// `'\b\nCommand requires higher access.\b\n'`) — the dispatcher's generic
/// denial, printed on `RESULT_NOT_ALLOWED` (`express.e:28400`). Legacy `\b`
/// renders `\r` on the wire — byte-pinned in `ae_tierd_fs.txt`. Generic by
/// design: any future ACS-gated command reuses this line, so it lives with
/// the dispatcher, not in file_list/wire.rs.
const HIGHER_ACCESS_LINE: &[u8] = b"\r\nCommand requires higher access.\r\n";
```

```rust
// dispatch arm, next to AlterFlags (~:802)
MenuCommand::FileStatus => {
    // Slice D8 (`FS`): internalCommandFS (`express.e:24871-24874`) gates on
    // ACS_CONFERENCE_ACCOUNTING and returns RESULT_NOT_ALLOWED emitting zero
    // bytes; the dispatcher tail (`:28400`) prints higherAccess() (`:3038`).
    // On the shipped board no account holds the right (captured every probe,
    // `ae_tierd_fs.txt`), so FS denies unconditionally here — the deny is
    // reachable without a domain gate (§10.4; see design §7). The granted
    // branch — fileStatus(0) (`:24141`) — is A11's surface; when a slice first
    // makes it reachable it introduces the ACS gate with BOTH branches live
    // (deferred; COMMAND_PARITY Q1, Option G).
    self.terminal.write(HIGHER_ACCESS_LINE).await?;
}
```

**Failure paths (§10.4):** the handler touches exactly one port (terminal);
`write(...).await?` propagates `T::Error` up the existing teardown path. No store, no
sub-prompt, no state mutated before the write → no partial-commit window, no
`unwrap`/`expect`/`panic!`. There is **no granted branch** to leave dead.
